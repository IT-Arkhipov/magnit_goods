## ✅ Исправление 1: Загрузка магазинов в инкогнито-режиме

**Коммит:** `ad7db06`

### Проблема
На странице "Товары" в инкогнито-режиме не отображался список магазинов, хотя магазины были выбраны на странице "Каталог".

### Причина
- **Race condition** - функция `loadStoresForComparison()` вызывалась до инициализации localStorage
- **Асинхронность** - логика выбора первого магазина выполнялась параллельно с загрузкой UI
- **JavaScript ошибка** - дублирование переменной `oldPrice`

### Решение
- Преобразован callback-based код в async/await для гарантии последовательности
- Автоматический выбор первого активного магазина при пустом localStorage
- Исправлена JavaScript ошибка с дублированием переменной

### Результат
✅ Магазины загружаются корректно (4 магазина)  
✅ Товары отображаются (5704 товара из 9 категорий)  
✅ Сравнение цен работает (4 колонки)  
✅ Нет JavaScript ошибок

## ✅ Исправление 2: Отображение процента увеличения цены

**Коммит:** `89798a1`

### Проблема
Товары с увеличением цены показывали промо-скидку от API вместо реального увеличения.
- Пример: кофе "Monarch Asian Selection" показывал +56%, хотя цена выросла только на +11%

### Причина
- Поле `discount_percent` содержит промо-скидку от API Магнита
- Поле `historical_discount_percent` содержит реальное историческое изменение цены
- Логика не различала эти два случая

### Решение
Использовать `historical_discount_percent` для товаров с `is_price_increase=True`, иначе использовать `discount_percent` от API.

### Результат
✅ Товары с увеличением цены показывают правильный процент

## ✅ Исправление 3: Расчёт процента изменения цены от предыдущего сканирования

**Коммит:** `7d9851a`

### Проблема
Процент изменения цены рассчитывался неправильно - использовались исторические данные за 14 дней вместо текущего изменения от предыдущего сканирования.

### Решение
При сканировании товаров:
1. Сравниваем текущую цену с предыдущей ценой в БД
2. Рассчитываем процент: `(новая - старая) / старая * 100`
3. Сохраняем в `discount_percent` и `is_price_increase`
4. На странице отображаем: (+X%) сиреневым для увеличения, (-X%) зелёным для уменьшения

### Результат
✅ 1016 товаров отображают изменение цены  
✅ Правильный расчёт процента от предыдущего сканирования

## ✅ Исправление 4: Сохранение процента изменения цены со знаком

**Коммит:** `6e7df59`

### Проблема
Использовали `abs(price_change_percent)`, что теряло информацию о знаке (увеличение vs уменьшение).

### Решение
- Сохраняем отрицательное значение (-X%) для уменьшения цены
- Сохраняем положительное значение (+X%) для увеличения цены
- При отображении проверяем знак и выбираем цвет:
  - `discount > 0` → сиреневый (+X%)
  - `discount < 0` → зелёный (-X%)

### Результат
✅ Правильное отображение цвета в зависимости от направления изменения цены
