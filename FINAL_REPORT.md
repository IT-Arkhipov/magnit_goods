## ✨ Ключевые особенности

### Гибридная архитектура

```
JSON (корневые категории)
    ↓
    ├─ Стабильная структура
    ├─ Быстрая инициализация
    └─ Контролируемая иерархия

API (подкатегории)
    ↓
    ├─ Актуальные данные
    ├─ Автоматическое обновление
    └─ Полная синхронизация
```

### Retry логика

```
Попытка 1 → Ошибка → Ожидание 2 сек
Попытка 2 → Ошибка → Ожидание 4 сек
Попытка 3 → Ошибка → Ожидание 8 сек
Попытка 4 → Ошибка → Пропуск категории
```

### Полная синхронизация

```
API возвращает категории
    ↓
Для каждой категории:
  ├─ Если новая → Добавить
  ├─ Если существует → Обновить
  └─ Если нет в API → Удалить
    ↓
Корневые категории защищены от удаления
```
