# Итоговый отчёт: Гибридная архитектура Product Scanning

**Дата:** 2026-04-14  
**Статус:** ✅ Архитектура переработана и готова к использованию

## Что было изменено

### 1. Стратегия загрузки категорий

**Было:**
- JSON содержал полный каталог (18 корневых + 123 подкатегории)
- Все категории загружались один раз при инициализации
- Подкатегории не обновлялись автоматически

**Стало:**
- JSON содержит только корневые категории (18 шт)
- Подкатегории получаются динамически из API
- Обновление каталога вручную через кнопку "Обновить каталог"
- Полная синхронизация при каждом обновлении

### 2. Новые endpoints

**POST /api/categories/update-catalog?store_code=992104**
- Основной endpoint для обновления каталога
- Получает все категории из API
- Добавляет новые подкатегории
- Обновляет существующие
- Удаляет устаревшие (полная синхронизация)
- Защищает корневые категории от удаления

**POST /api/categories/load-from-json** (обновлен)
- Теперь загружает только корневые категории
- Подкатегории из JSON игнорируются
- Используется при первой инициализации

### 3. Обновленная логика

**load_catalog_from_json.py:**
- Читает только корневые категории из JSON
- Игнорирует подкатегории (subcategories)
- Возвращает: `{"scanned": 18, "added": X, "updated": Y}`

**catalog_scanner.py (метод scan_categories):**
- Получает ВСЕ категории из API
- Добавляет новые подкатегории
- Обновляет существующие (name, url, product_count)
- Удаляет подкатегории, которых нет в API
- Защищает корневые категории (parent_id=None)
- Возвращает: `{"scanned": N, "added": X, "updated": Y, "deleted": Z}`

## Workflow

### Инициализация (один раз)

```
1. POST /api/categories/load-from-json
   ↓ Загружаются 18 корневых категорий из JSON
   
2. POST /api/categories/update-catalog?store_code=992104
   ↓ Получаются подкатегории из API
   ↓ Добавляются в БД с parent_id, указывающим на корневую категорию
   
3. POST /api/categories/update-tracking
   ↓ Выбираются категории для отслеживания (is_tracked=True)
   
4. POST /api/catalog/scan-all-stores
   ↓ Сканируются товары для отслеживаемых категорий
```

### Обновление каталога (вручную)

```
POST /api/categories/update-catalog?store_code=992104
↓
Получаются все категории из API
↓
Для каждой категории:
  - Если новая: добавить
  - Если существует: обновить
  - Если нет в API: удалить
↓
Результат: {"scanned": N, "added": X, "updated": Y, "deleted": Z}
```
