### Технические детали

**База данных**
- SQLite: `src/data/magnit.db`
- Таблицы: stores, categories, products, price_history, scan_jobs
- Всего категорий: 147 (24 корневые, 123 подкатегории)

**API Endpoints (Каталог)**
- `GET /api/categories/tree` — дерево категорий (200 OK)
- `GET /api/categories?tracked=true` — отслеживаемые категории (200 OK)
- `POST /api/categories/update-tracking` — сохранить выбор (200 OK)

**Веб-интерфейс**
- URL: http://localhost:8000/catalog
- Статус: Полностью функционален
- Поддерживаемые браузеры: Chrome, Firefox, Safari, Edge

### Известные проблемы и решения

| Проблема | Решение | Статус |
|----------|---------|--------|
| API возвращал только корневые категории при фильтре `tracked=true` | Исправлен эндпоинт `/api/categories` — условие `parent_id` теперь не применяется при наличии фильтра `tracked` | ✅ Решено |
| Счётчик показывал 0 вместо 21 при загрузке | Добавлена задержка перед обновлением состояния для синхронизации с DOM | ✅ Решено |
| Сервер падал при timeout в bash команде | Создан batch-файл для запуска сервера в отдельном окне | ✅ Решено |
