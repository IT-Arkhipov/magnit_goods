# Руководство пользователя

Веб-приложение для мониторинга цен в магазинах «Магнит». Позволяет добавлять магазины, сканировать каталог товаров, отслеживать изменения цен и сравнивать цены между магазинами.

## Навигация

Верхняя панель на всех страницах:

| Раздел | URL | Назначение |
|--------|-----|-----------|
| **Магазины** | `/` | Добавление и управление магазинами, выбор активного магазина |
| **Каталог** | `/catalog` | Дерево категорий, включение отслеживания, сканирование товаров |
| **Товары** | `/products` | Список товаров с фильтрами, два режима отображения цен, сравнение по магазинам, список покупок |
| **Список покупок** | `/shopping-list` | Корзина выбранных товаров с проверкой наличия |
| **Скидки** | — | Раздел «скоро будет» (заглушка в навигации) |
| **Задания** | `/jobs` | Фоновые задания: статусы, прогресс, отмена |

## Рабочий процесс

### Шаг 1. Добавление магазинов (`/`)

На главной странице — раздел «Добавление магазинов»:

1. Введите **город** (обязательно) и/или **улицу** в форму.
2. Выберите типы магазинов для поиска (Магнит, Экстра, М.Косметик и др.).
3. Нажмите **«Сканировать»** — сервер обратится к API Магнита и покажет preview найденных магазинов.
4. В таблице preview отметьте нужные галочками (магазины, уже добавленные в базу, помечены и недоступны для выбора).
5. Нажмите **«Добавить выбранные (N)»** — выбранные магазины сохранятся в базу.

Ниже — **«Список магазинов в базе»**: можно выбрать активный магазин (обновит `.env`), удалить магазины (выделив галочками → «Удалить выбранные»), найти магазин по адресу.

> 🔴 **Активный магазин** (в `.env`) используется scheduler-ом для автоматических заданий. Переключайте через выбор магазина в списке, не редактируя `.env` вручную.

### Шаг 2. Настройка каталога (`/catalog`)

Страница с деревом категорий и управлением отслеживанием:

1. **Загрузка категорий:** при первом запуске нажмите **«Обновить каталог»** — сервер получит актуальное дерево категорий из API Магнита (полная замена с сохранением флагов `is_tracked`).
2. **Выбор отслеживаемых категорий:** отметьте дочерние категории (не корневые) галочками.
3. Нажмите **«Сохранить выбор»** — выбранные категории получат `is_tracked=True`.
4. **«Сканировать товары»** — запускает фоновое сканирование товаров из отслеживаемых категорий. Во время сканирования появляется кнопка **«Остановить»**.
5. После завершения сканирования флаг «Новая цена» автоматически включается для страницы товаров.

Внизу страницы — **«Управление базой данных товаров»**:
- **Очистить по выбранным категориям** — удаляет товары только для отмеченных категорий.
- **Очистить всю базу** — удаляет все товары (с подтверждением).

> Отслеживаемые (`is_tracked`) категории участвуют в ежедневном автоматическом обновлении цен (scheduler, 08:00) и еженедельном сканировании каталога (воскресенье 06:00). См. `docs/DEPLOYMENT.md`.

### Шаг 3. Просмотр товаров (`/products`)

Главная страница анализа товаров.

#### Фильтры и поиск
- **Выбор магазина** — выпадающий список активных магазинов.
- **Категории** — выпадающий список с чекбоксами, кнопки «Выбрать все» / «Снять все».
- **Поиск** — текстовое поле (поиск по названию товара, работает с кириллицей).
- **Цена** — фильтр по минимальной/максимальной цене.
- **Сортировка** — по названию / цене / дате последнего скана.

#### Статистика товаров
Панель над списком показывает: всего товаров, в наличии, со снижением цены, с повышением цены, дата последнего обновления.

#### Два режима отображения изменения цен
Чекбокс **«Новая цена»** (справа в фильтрах) переключает режимы:

| Режим | Чекбокс | Что показывает | Тултип при наведении |
|-------|---------|----------------|----------------------|
| **«Новая цена»** (по умолчанию) | ☑ включён | Изменение цены **от предыдущего сканирования** | Остаток товара + дата последнего скана |
| **«Последнее изменение»** | ☐ выключен | **Последнее реальное изменение** цены (даже если после этого было несколько сканов без изменений) | Предыдущая цена + дата изменения |

**Цветовая индикация:**
- 🟢 **зелёная стрелка `↓15%`** — цена **снизилась** на 15% (`price_change_percent > 0`).
- 🟣 **фиолетовая стрелка `↑11.8%`** — цена **повысилась** на 11.8% (`price_change_percent < 0`).
- Без стрелки — цена не изменилась в текущем режиме.

Состояние чекбокса сохраняется в `localStorage` и восстанавливается при перезагрузке. После сканирования товаров флаг автоматически устанавливается в `true` (чтобы сразу увидеть свежие изменения).

#### Сравнение цен по магазинам
1. В верхней панели **«Сравнение товаров по магазинам»** отметьте 2+ магазина.
2. Нажмите **«Применить выбор»**.
3. У каждого товара появится кнопка **«Сравнить по магазинам»** (если выбран 1 магазин) или **«Сравнить»** (если 2+).
4. Откроется модальное окно с ценами товара во всех выбранных магазинах, остатками и изменениями.

#### Список покупок
На странице товаров (или в навигации → «Список покупок»):
- У каждого товара есть кнопка добавления в список покупок.
- Список сохраняется в `localStorage`.
- На `/shopping-list` — проверка наличия товаров: «не найден в базе» / «нет в наличии» с возможностью удалить недоступные позиции.
- **Экспорт в текст** — копирование списка в буфер обмена.
- **Очистить весь список.**

### Шаг 4. Отслеживание заданий (`/jobs`)

Страница со списком фоновых заданий (сканирование товаров, обновление цен, обновление каталога).

- **«Обновить список»** — перезагрузить таблицу заданий.
- Для каждого задания показывается: тип (`prices`/`catalog`/`scan_all_stores`), статус, прогресс, кол-во товаров (просканировано/добавлено/обновлено), время начала/завершения, ошибка.
- **Статусы:** `pending` (в очереди), `running` (выполняется), `completed` (завершено), `failed` (ошибка), `cancelled` (отменено).
- Для активных заданий (`pending`/`running`) доступна кнопка **отмены** — установка статуса `cancelled`. Реальная остановка происходит при следующей итерации сканирования (проверка `ScanJob.status == "cancelled"`).

> Сканирование 5000+ товаров может занимать 10–30 минут из-за rate limiting (0.5s между запросами к API Магнита). Поэтому операции запускаются в фоне, а прогресс виден на странице заданий. См. `docs/scan_jobs.md`.

## Автоматические задания (scheduler)

Если в `.env` задан `STORE_CODE`, при старте сервера запускаются автоматические задания:

| Расписание | Задание |
|-----------|---------|
| Ежедневно 08:00 | Обновление цен для отслеживаемых категорий |
| Воскресенье 06:00 | Полное сканирование каталога (категории + товары) |

Результаты заданий видны на странице «Задания». См. `docs/DEPLOYMENT.md`.

## Частые вопросы

**Почему сканирование идёт долго?**
Rate limiting 0.5s между запросами + случайная пауза 0.1–0.5s для защиты от бана. Для 5000 товаров это ~50 минут. Это нормально.

**Почему цена показывается без стрелки, хотя раньше менялась?**
В режиме «Новая цена» (по умолчанию) стрелка показывается только если цена изменилась **с последнего скана**. Если цена была 120₽ и осталась 120₽ — стрелки не будет. Переключите чекбокс «Новая цена» в режим «Последнее изменение» — увидите последнее реальное изменение.

**Как увидеть историю цены товара?**
На странице товаров нажмите на товар — откроются детали с историей цены по дням (endpoint `GET /api/products/{id}/history`). История хранится 30 дней.

**Почему некоторые магазины не находятся при поиске?**
Поиск через API Магнита. Если у магазина не определился город (через regex-парсинг адреса) — он пропускается. Попробуйте уточнить запрос (город + улица).

**Можно ли открыть товар на magnit.ru сразу с нужным магазином?**
Да — у каждого товара есть ссылка-цена, которая открывает товар в браузере через Playwright с автоматическим выбором магазина (`GET /open-product-in-browser`). Требуется установленный Chromium: `playwright install chromium`.
